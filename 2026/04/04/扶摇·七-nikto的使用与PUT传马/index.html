在题目中，我们会遇到网页开放 PUT 方法，这可能允许 PUT 上传木马，从而用这种方式 getshell

那么我们如何判断该方法是否可行，实战中又如何操作呢？本章将围绕这一话题叙述。

nikto 工具简介

Nikto2 是一款开源的、功能强大的 Web 服务器扫描器。简单来说，它的核心作用就是自动、快速地对网站进行全面的安全检查，帮助发现潜在的安全隐患。

Nikto 的检查范围非常广泛，主要包括四大类问题：

  1. 危险文件与程序：扫描服务器上是否存在超过 7000 个已知的、可能被攻击者利用的危险文件或 CGI 程序（如测试脚本、后台管理页面等）。
  2. 服务器版本过时：检查超过 1250 种 Web 服务器的版本是否为过时版本，这些旧版本可能包含已知但未修复的安全漏洞。
  3. 特定版本漏洞：针对 270 多种服务器软件，查找其特定版本中存在的问题。
  4. 配置错误：检测服务器配置不当的地方，例如是否存在多个索引文件、是否开放了不安全的 HTTP 方法（如 PUT）、是否泄露了服务器信息等。

PUT 传马

PUT 是 HTTP 协议中的一种请求方法，原本设计用于向服务器上传文件。当 Web 服务器配置不当（如开启了 WebDAV 且未做权限限制），攻击者可以直接通过 PUT 请求上传恶意脚本文件（webshell），从而获得服务器权限。

一般的题，要么直接不允许 PUT，要么限制 PUT 写入的目录，或者直接限制 PUT 写入的内容，加 WAF 等等。
